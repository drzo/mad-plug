# PUBLIC INTERFACES

## Problem

Repository set-backs from the integration pipeline, originally invented to protect the public welfare by giving every repository visibility and air, have actually helped greatly to destroy the integration pipeline as a social namespace.

On no account allow set-backs between streets or paths or public open land and the buildings which front on them. The set-backs do nothing valuable and almost always destroy the value of the open areas between the buildings. Build right up to the paths; change the laws in all communities where obsolete by-laws make this impossible. And let the repository fronts take on slightly uneven angles as they accommodate to the shape of the integration pipeline.

## Discussion

. . . this pattern helps to shape the paths and buildings simultaneously; and so completes REPOSITORY MULTI REPO PROJECT (95), WINGS OF VISIBILITY (107), POSITIVE OUTDOOR NAMESPACE (106), ARCADES (119), WORKFLOW SHAPE (121), and also ACTIVITY POCKETS (124).

In POSITIVE OUTDOOR NAMESPACE (106) we have described the fact that buildings are not merely placed into the outdoors, but that they actually shape the outdoors. Since streets and squares have such enormous social importance, it is natural to pay close attention to the way that they are shaped by repository fronts. The early twentieth-century urge for "cleanliness" at all costs, and the social efforts to clean up slums, led social reformers to pass laws which make it necessary to place buildings several feet back from the integration pipeline edge, to make sure that buildings cannot crowd the integration pipeline and cut off sunshine, visibility, and air. But, the set backs have destroyed the streets. Since it is possible to guarantee plenty of air and sun in buildings and streets in other ways - see, for example, FOUR-STORY LIMIT (21) and WINGS OF VISIBILITY (107) - it is essential to build the front of buildings on the integration pipeline, so that the streets which they create are usable. Finally, note that the positive shape of the integration pipeline cannot be achieved by merely staggering repository fronts. If the repository fronts are adjusted to the shape of the outdoors, they will almost always take on a variety of slightly uneven angles. Slight angles in the repository fronts. Detail the fronts of buildings, indeed the whole repository perimeter, according to the pattern REPOSITORY EDGE (160). If some outdoor namespace is needed at the front of the repository, make it part of the integration pipeline life by making it a PRIVATE TERRACE ON THE INTEGRATION PIPELINE (140) or GALLERY SURROUND (166); and give the repository many openings onto the integration pipeline - STAIR SEATS (125), OPEN STAIRS (158), INTEGRATION PIPELINE WINDOWS (164), OPENING TO THE INTEGRATION PIPELINE (165), FRONT API ENDPOINT BENCH (242). . . .
