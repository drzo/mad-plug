# Domain Transformations Reference

## Overview

Each pattern can be viewed through 6 dimensional perspectives (dim0-dim5), enabling domain-specific applications.

## Dimension Mappings

| Dimension | Domain | Application |
|-----------|--------|-------------|
| dim0 | Archetypal | Abstract with `{{placeholders}}` |
| dim1 | Template | Generic templates |
| dim2 | Physical | Architecture, spatial design |
| dim3 | Social | Organizations, communities |
| dim4 | Conceptual | Knowledge, systems |
| dim5 | Interpersonal | Awareness, consciousness |

## Example: Pattern 107 (Wings of Light)

| Domain | Transformation |
|--------|---------------|
| Physical | Narrow building wings for natural light |
| Social | Flat hierarchies for information transparency |
| Conceptual | Modular code with visible core logic |
| Interpersonal | Mental structures for awareness |

## Example: Pattern 1 (Independent Regions)

| Domain | Transformation |
|--------|---------------|
| Physical | Geographic regions with boundaries |
| Social | Autonomous teams with clear ownership |
| Conceptual | Microservices with bounded contexts |
| Interpersonal | Personal boundaries and identity |

## Accessing Transformed Patterns

```
patterns/apl0/dim{N}/cat{C}/seq{SS}/apl{NNN}.md
```

Examples:
- Physical Pattern 107: `dim2/cat2/seq17/apl107.md`
- Social Pattern 1: `dim3/cat1/seq01/apl001.md`
- Conceptual Pattern 52: `dim4/cat1/seq08/apl052.md`
