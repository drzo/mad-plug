# Time Crystal Neuron Model Generator

This skill constructs a **Time Crystal Generalized Neuron Model** for a given context. It translates the hierarchical, multi-scale temporal patterns found in the Nanobrain project (Figure 6.14) into a formal neural network architecture specification.

Use this skill to design complex neural models that incorporate principles of nested periodicity, phase coupling, and fractal organization. It is ideal for modeling systems with inherent temporal hierarchies, such as language, finance, or biological processes.

## Core Concepts

The model is based on three pillars:

1.  **Time Crystal Neuron (from Nanobrain)**: A biological neuron model where components oscillate at different, harmonically related frequencies. See the reference image in `templates/time-crystal-neuron.PNG`.
2.  **sys-n Framework**: A deterministic state transition model for cyclic systems. The slow oscillators in the neuron map to Universal Sets (global state), and fast oscillators map to Particular Sets (local processing).
3.  **nn (Torch) Framework**: A modular neural network library. The hierarchical structure of the neuron is mapped to nested `nn.Sequential` and `nn.Concat` containers.

For a detailed explanation of the theory and component mappings, see `references/theory.md` and `references/components.md`.

## Workflow

The process of creating and using a time crystal neuron model involves three main steps:

1.  **Generate the Model**: Use `generate_tcn_model.py` to create a model specification for your specific context (e.g., "language processing"). This script maps your domain to the neuron's components and generates a JSON definition, a Lua/Torch `nn` architecture, and a `sys-n` state specification.

2.  **Visualize the Architecture**: Use `visualize_tcn.py` to generate diagrams from the JSON model. This helps in understanding the model's structure, temporal hierarchy, and state dynamics.

3.  **Implement and Train**: Use the generated `.lua` file as a starting point for a Torch7 implementation. The `sys-n` specification can guide the implementation of the state transition logic.

## Scripts

### 1. `generate_tcn_model.py`

This is the primary script for creating a model specification.

**Usage**:
```bash
python /home/ubuntu/skills/time-crystal-neuron/scripts/generate_tcn_model.py \
    --context "your domain context" \
    --structure <a,b,c,d> \
    --output model.json --nn --sysn
```

| Argument | Description |
|---|---|
| `--context` | The application domain (e.g., "audio processing", "robotics"). Required. |
| `--structure` | The `[a,b,c,d]` hierarchical structure. Defaults to `3,4,3,3`. |
| `--output` | The path to save the output JSON model file. |
| `--nn` | If set, generates a `.lua` file with the Torch `nn` architecture. |
| `--sysn` | If set, generates a `.md` file with the `sys-n` state specification. |

**Example**:
```bash
# Generate a model for financial market analysis
python /home/ubuntu/skills/time-crystal-neuron/scripts/generate_tcn_model.py \
    --context "financial markets" \
    --output financial_tcn.json --nn --sysn
```

### 2. `visualize_tcn.py`

This script generates visualizations from a model's JSON file.

**Usage**:
```bash
# Generate a hierarchy diagram
python /home/ubuntu/skills/time-crystal-neuron/scripts/visualize_tcn.py model.json -o diagram.png

# Generate a state transition timeline
python /home/ubuntu/skills/time-crystal-neuron/scripts/visualize_tcn.py model.json --timeline -o timeline.png

# Generate a reference diagram of temporal scales
python /home/ubuntu/skills/time-crystal-neuron/scripts/visualize_tcn.py --scales -o scales.png
```

| Argument | Description |
|---|---|
| `model.json` | Path to the JSON model file generated by the first script. |
| `-o` | The output path for the generated PNG image. |
| `--timeline` | Generate a state transition timeline instead of a hierarchy diagram. |
| `--scales` | Generate a reference chart of the 9 temporal scales. Does not require a model file. |
